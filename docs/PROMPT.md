## What this is

A personal, reusable **monorepo PWA template** вЂ” mobile-first, production-ready,
fully containerized. Designed as a starting base for future projects.

## Key architectural decisions

- **Single Traefik entry point** вЂ” all traffic enters via `https://localhost:4443` (local) or `https://$DOMAIN` (prod). Subpaths: `/api/` в†’ backend, `/auth/` в†’ Keycloak, `/pgadmin/` в†’ pgAdmin, `/` в†’ frontend. Local uses ports `4443` (HTTPS) and `8090` (HTTP redirect) because port 80/443 may be in use on the host.
- **TLS** вЂ” self-signed cert for local (auto-generated by `run.sh certs`), Let's Encrypt for prod. Toggle via `TLS_MODE=local|letsencrypt` in `.env`.
- **Auth flow** вЂ” Angular в†’ Keycloak (handles Google/Facebook OAuth) в†’ Keycloak JWT в†’ FastAPI verifies via JWKS. Backend has zero Google/Facebook dependency.
- **Two databases, one Postgres instance** вЂ” `app_db` (Alembic-managed) and `keycloak_db` (Keycloak-managed). Both on the `postgres:17` container.
- **No NgModules** вЂ” Angular v19 fully standalone components throughout.
- **SCSS + BEM** вЂ” Tailwind was evaluated and rejected due to Ionic conflicts. BEM for custom components, Ionic CSS variables for Ionic theming, `::part()` for shadow DOM access.

## How to run

```bash
cp .env.example .env          # fill in passwords and secrets
./run.sh dev                  # starts all services with hot reload
./run.sh logs [service]       # tail logs
./run.sh db:migrate           # run Alembic migrations
./run.sh db:revision "msg"    # create new migration
./run.sh stop                 # stop everything
./run.sh help                 # full command reference
```

To run Storybook (from `frontend/`):
```bash
npm run storybook      # dev server with hot reload
```

Services after `./run.sh dev`:
- `https://localhost:4443/` вЂ” Angular PWA
- `https://localhost:4443/api/v1/docs` вЂ” FastAPI Swagger
- `https://localhost:4443/auth/` вЂ” Keycloak admin
- `https://localhost:4443/pgadmin/` вЂ” pgAdmin
- `https://localhost:8080/` вЂ” Traefik dashboard

First run: browser will warn about the self-signed cert вЂ” type `thisisunsafe` (Chrome) or accept the risk (Firefox).

## Theming

`ThemeService` is injected at app root. Two signals: `scheme` and `accent`. Both persist to `localStorage`.

- Call `theme.setScheme('light' | 'dark' | 'system')` and `theme.setAccent('clay' | 'moss' | 'dune' | 'slate' | null)`
- CSS is in `src/theme/variables.scss` вЂ” 8 accent mixins Г— dark/light, applied via `body.dark` / `body[data-accent]`
- The `SettingsPage` exposes the full UI (3-way segment + 5 swatches)

## Shared components

All 22 components are in `shared/components/` and exported from `shared/index.ts`. Import from there:

```ts
import { AvatarComponent, CardComponent, SectionComponent } from '../../shared';
```

Storybook at `http://localhost:6006` вЂ” run via `./run.sh storybook`.

## Keycloak realm

Pre-configured realm `pwa` is imported on container startup from `infra/keycloak/realm-export.json`.

- Client: `pwa-frontend` (public, PKCE S256, redirect URIs for `https://localhost:4443/*`, `http://localhost:4200/*`, `capacitor://localhost/*`)
- Roles: `user` (default), `admin`
- Login theme: `pwa` (custom FTL + CSS in `infra/keycloak/themes/pwa/login/`)
- Identity Providers: Google and Facebook (disabled by default вЂ” enable after adding credentials to `.env`)
- Social IdP redirect URIs (set in Google/Facebook console):
  - `https://$DOMAIN/auth/realms/pwa/broker/google/endpoint`
  - `https://$DOMAIN/auth/realms/pwa/broker/facebook/endpoint`

**Known quirk**: If the realm already exists in the DB, Keycloak silently skips re-import.
To apply realm changes to an existing instance: use `kcadm.sh update realms/pwa -s key=value` or wipe the postgres volume.

**FTL gotcha**: Keycloak 26 FreeMarker templates use auto-escaping вЂ” `?html` is not allowed. Write plain `${}` expressions.

## Conventions summary

Full rules in `docs/CONVENTIONS.md`. Key points:

- **Git**: Conventional Commits вЂ” `feat(scope): summary`
- **Angular files**: kebab-case, typed suffix (`.page.ts`, `.component.ts`, `.service.ts`, etc.)
- **Angular components**: standalone only, `app-` selector prefix, one per file
- **Styling**: SCSS + BEM вЂ” `.block__element--modifier`. Max 2 nesting levels. Ionic CSS vars for theming. `::part()` for shadow DOM.
- **NgRx**: store for shared/server data only. Actions describe events (past tense). One action group per feature. Effects own all side effects.
- **Python**: Black (88) + Ruff. Type hints everywhere. Async-only DB. Thin endpoints, logic in services.
- **DB**: plural snake_case tables, UUID PKs, `created_at`/`updated_at` on every table. Alembic only (no `create_all` in prod).
- **Docker**: multi-stage Dockerfiles (`base` в†’ `development` в†’ `production`). Bind mounts in local only.
- **Storybook**: stories only for `shared/components/`. Always include `tags: ['autodocs']`. Title pattern: `Shared/<Category>/<ComponentName>`. Co-located with component file.

**If the user asks you to violate a convention, flag it and ask for double confirmation before proceeding.**

## Known gotchas (check before debugging)

| Symptom | Root cause | Fix |
|---------|-----------|-----|
| Navigating to any tab redirects to home | `tabs.page.ts` missing `<ion-router-outlet />` inside `<ion-tabs>` | Add `IonRouterOutlet` to imports + template |
| All API calls return 401 | Keycloak JWT `iss` uses public URL; backend verifies against internal URL | Set `KEYCLOAK_PUBLIC_URL` in `.env`; `config.py` uses it for `KEYCLOAK_ISSUER` |
| "Sorry internal server error" after social login | Custom Keycloak theme missing `login-update-profile.ftl` | Create the FTL using `@layout.page` macro |
| `::part()` in component SCSS has no effect | Angular emulated encapsulation breaks `::part()` selectors | Move `::part()` rules to `src/global.scss` |
| Redirect loop on login | Auth guards using `router.navigate() + return false` | Return `router.createUrlTree([...])` from guards |
| Ionic components have no styling / differ from Storybook | `provideIonicAngular()` missing from `app.config.ts` | Add `provideIonicAngular()` to providers |
| Card variants are transparent in dark mode | `background: rgba(tint)` replaces card surface color | Use CSS multiple backgrounds to layer tint over `--ion-card-background` |
