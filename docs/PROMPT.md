# PWA Template — Project Context Primer

Read this file at the start of every session before making any changes.

## What this is

A personal, reusable **monorepo PWA template** — mobile-first, production-ready,
fully containerized. Designed as a starting base for future projects.

---

| Layer          | Technology                        |

## Key architectural decisions

- **Single Traefik entry point** — all traffic enters via `https://localhost:4443` (local) or `https://$DOMAIN` (prod). Subpaths: `/api/` → backend, `/auth/` → Keycloak, `/pgadmin/` → pgAdmin, `/` → frontend. Local uses ports `4443` (HTTPS) and `8090` (HTTP redirect) because port 80/443 may be in use on the host.
- **TLS** — self-signed cert for local (auto-generated by `run.sh certs`), Let's Encrypt for prod. Toggle via `TLS_MODE=local|letsencrypt` in `.env`.
- **Auth flow** — Angular → Keycloak (handles Google/Facebook OAuth) → Keycloak JWT → FastAPI verifies via JWKS. Backend has zero Google/Facebook dependency.
- **Two databases, one Postgres instance** — `app_db` (Alembic-managed) and `keycloak_db` (Keycloak-managed). Both on the `postgres:17` container.
- **No NgModules** — Angular v19 fully standalone components throughout.
- **SCSS + BEM** — Tailwind was evaluated and rejected due to Ionic conflicts. BEM for custom components, Ionic CSS variables for Ionic theming, `::part()` for shadow DOM access.

## How to run

```bash
cp .env.example .env          # fill in passwords and secrets
./run.sh dev                  # starts all services with hot reload
./run.sh logs [service]       # tail logs
./run.sh db:migrate           # run Alembic migrations
./run.sh db:revision "msg"    # create new migration
./run.sh stop                 # stop everything
./run.sh help                 # full command reference
```

Services after `./run.sh dev`:
- `https://localhost:4443/` — Angular PWA
- `https://localhost:4443/api/v1/docs` — FastAPI Swagger
- `https://localhost:4443/auth/` — Keycloak admin
- `https://localhost:4443/pgadmin/` — pgAdmin
- `https://localhost:8080/` — Traefik dashboard

First run: browser will warn about the self-signed cert — type `thisisunsafe` (Chrome) or accept the risk (Firefox).

## Frontend structure

```
src/app/
├── core/
│   ├── auth/            KeycloakService wrapper, authGuard, LoginPage
│   └── interceptors/    auth (token injection), error (401 → re-login)
├── features/
│   ├── tabs/            TabsPage — bottom tab navigation shell
│   ├── home/            HomePage — welcome screen with logout
│   └── example/         Full CRUD feature (list + detail pages, NgRx store, API service)
├── shared/              (empty — add reusable components here)
└── store/               Root NgRx registration (rootReducers, rootEffects)
```

Entry point: `src/main.ts` → `app.config.ts` (providers: router, store, Keycloak init, HTTP interceptors).

## Backend structure

```
app/
├── main.py              FastAPI factory, CORS, lifespan (create_all for dev)
├── api/v1/
│   ├── router.py        Aggregates all endpoint routers
│   └── endpoints/
│       ├── health.py    GET /api/v1/health (public)

│       └── ws.py        WebSocket /api/v1/ws/{channel} (token via query param)
├── core/
│   ├── config.py        pydantic-settings — all config from .env
│   ├── security.py      JWT verification via Keycloak JWKS (with cache + rotation)
│   └── dependencies.py  get_db, get_current_user, require_role
├── db/
│   ├── base.py          SQLAlchemy DeclarativeBase
│   └── session.py       Async engine + session factory
├── models/example.py    ExampleItem ORM model
└── schemas/example.py   Pydantic schemas (Base, Create, Update, Response)
```

Migrations: `backend/alembic/` — `env.py` reads `settings.DATABASE_URL` and auto-imports all models.

## Keycloak realm

Pre-configured realm `pwa` is imported on container startup from `infra/keycloak/realm-export.json`.

- Client: `pwa-frontend` (public, PKCE, redirect URIs for localhost + Capacitor)
- Roles: `user` (default), `admin`
- Identity Providers: Google and Facebook (disabled by default — enable after adding credentials to `.env`)
- Social IdP redirect URIs (set in Google/Facebook console):
  - `https://$DOMAIN/auth/realms/pwa/broker/google/endpoint`
  - `https://$DOMAIN/auth/realms/pwa/broker/facebook/endpoint`

## Conventions summary

Full rules in `docs/CONVENTIONS.md`. Key points:

- **Git**: Conventional Commits — `feat(scope): summary`
- **Angular files**: kebab-case, typed suffix (`.page.ts`, `.component.ts`, `.service.ts`, etc.)
- **Angular components**: standalone only, `app-` selector prefix, one per file
- **Styling**: SCSS + BEM — `.block__element--modifier`. Max 2 nesting levels. Ionic CSS vars for theming. `::part()` for shadow DOM.
- **NgRx**: store for shared/server data only. Actions describe events (past tense). One action group per feature. Effects own all side effects.
- **Python**: Black (88) + Ruff. Type hints everywhere. Async-only DB. Thin endpoints, logic in services.
- **DB**: plural snake_case tables, UUID PKs, `created_at`/`updated_at` on every table. Alembic only (no `create_all` in prod).
- **Docker**: multi-stage Dockerfiles (`base` → `development` → `production`). Bind mounts in local only.

**If the user asks you to violate a convention, flag it and ask for double confirmation before proceeding.**
